# Glossary

Short definitions for Pollux's core terms.

## Context caching

Uploading shared content once and reusing it across prompts.

## Fan-out

One source -> many prompts.

## Fan-in

Many sources -> one prompt.

## Broadcast

Many sources x many prompts.

## Source patterns

Collective term for fan-out, fan-in, and broadcast.

## Deferred mode

Reserved option for provider async batch delivery APIs; intentionally disabled
in v1.0.

## Capability transparency

Pollux surfaces provider differences explicitly instead of silently degrading
or pretending feature parity.
